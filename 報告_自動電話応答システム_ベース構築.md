# 報告資料：Azure を用いた自動電話応答システムのベース構築

## 1. 趣旨・位置づけ

Azure を利用し、**自動電話応答システムのベース（プロトタイプ基盤）**を構築した。本資料は、その内容・デモ想定・技術観点・課題・今後計画、および関連するライセンス検討事項を観点整理したものである。

---

## 2. 概要（何をしたか）

- **Azure を前提とした受電〜応答フローの土台**を用意した。
- 通話シナリオに沿った**デモで動作確認できる状態**まで持ち込んだ。
- AI の利用範囲を意図的に絞り、**本丸を「対応 Agent（判断・分岐の集約）」**に置いている。

---

## 3. デモで想定している問い合わせ例

デモ時に見せる想定発話の例は次のとおり。

| #   | お客様側の発話イメージ   |
| --- | ------------- |
| 1   | 賞味期限を教えて      |
| 2   | 配送遅延について教えて   |
| 3   | 営業部の佐藤さんにつないで |

FAQ 的な問い合わせと、担当者への取り次ぎの両方がデモで説明できる想定とする。

---

## 4. 技術観点：AI の使いどころ

| 用途       | 補足                    |
| -------- | --------------------- |
| **STT**  | 発話のテキスト化              |
| **TTS**  | ガイダンス・応答の音声化          |
| **要約生成** | 通話内容等の要約（保存・連携の前提として） |

**ルーティング判断・会話方針の「本丸」は対応 Agent 側**に置き、上記の生成系はその周辺処理として位置づけている。

---

## 5. 課題（現時点）

| 観点        | 内容                                                           |
| --------- | ------------------------------------------------------------ |
| **STT**   | **認識精度**。実運用に耐えるかは継続的な評価が必要。                                 |
| **Agent** | **設計と実装**。意図理解・分岐・安全な応答の責務をどう切るかが今後の焦点。                      |
| **汎用性**   | 現状は **CSV 内キーワードへのヒット有無**による分岐に限定されており、**汎用的な意図理解にはなっていない**。 |

---

## 6. 今後の予定

- **次週以降**、**LangChain 等を用いた Agent 基盤**への移行を検討・着手する。
- キーワードマッチ中心から、**エージェントによる応答・ツール呼び出し**へ拡張し、汎用性と保守性を高める方向とする。

（詳細スケジュール・スコープは別途すり合わせする。）

---

## 7. 滞留・相談事項（Teams Phone ライセンス）

**状況の要約**

- IIJ 経由で **Teams Phone** のライセンス購入を進めようとしたところ、**グローバル管理者などの特権ロール**を求められる提示があり、**いったん保留**としている。
- **最小権限に絞れるか**を IIJ に確認しつつ、**同様に Teams Phone ライセンスを導入している他社では、実務上どの程度の権限が要求されているか**もあわせて確認している。
- **IIJ 側の回答次第**で、**乗り換え**や **Microsoft との直接契約（年間契約等）**も選択肢に含めて検討する可能性がある。その際は **別途相談**する。
